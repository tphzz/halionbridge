# 064 Pitch Envelope Implementation Regression

This probe follows the hand-authored pitch-envelope diagnostics `061` through
`063`. It is generated by `halionbridge convert sfz` and validates the current
implementation candidate:

- `pitcheg_depth` in cents maps to HALion `Pitch.EnvAmount` in semitones.
- zero-attack full-depth pitch state starts at envelope level `1`.
- `pitcheg_decay` maps to a HALion pitch-envelope decay segment with duration
  `0.30 * pitcheg_decay` and Lua curve `-0.6`.

## Files

- `source/*.sfz` are same-number sforzando references.
- `source/samples/saw_A3_single_cycle.wav` is copied from probe `063`.
- `plugnscript/halionbridge_sfz_pitch_env_generator.cxx` emits one long note 57
  at velocity 127.
- `halionbridge-build/*.lua` are generated by `halionbridge convert sfz`.
- `halionbridge-build/*.vstpreset` are built through HALion and ready to load.

The SFZ references intentionally omit `end=` and use only `loop_mode`,
`loop_start`, and `loop_end`.


## Measurement Loop

1. Put Blue Cat Plug'n Script immediately before sforzando or HALion.
2. Load `plugnscript/halionbridge_sfz_pitch_env_generator.cxx`.
3. Compare same-number pairs from `000` through `007`.
4. Report which decay durations generalize and whether the negative-depth case
   behaves with the expected direction.

## Cases

- `000`: no pitch envelope baseline.
- `001`: one-second attack control.
- `002`: zero-attack full-depth hold.
- `003`: `pitcheg_decay=0.25`.
- `004`: `pitcheg_decay=0.5`.
- `005`: `pitcheg_decay=1.0`.
- `006`: `pitcheg_decay=2.0`.
- `007`: `pitcheg_depth=-1200`, `pitcheg_decay=1.0`.
