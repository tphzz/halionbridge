# 034 Pan Implementation Regression

This probe validates the converter implementation of SFZ `pan` after probe
`033` identified HALion `Amp.Pan` as the matching target.

The generated converter output should set:

```lua
Zone["Amp.Pan"] = sfz_pan
```

HALion's default `Amp.PanLaw` is intentionally left unchanged. Probe `033`
showed that changing pan law made the result worse, matching the expectation
that sforzando and HALion both use a -3 dB center pan law for this case.

## Files

- `source/*.sfz` are the sforzando references copied from probe `033`.
- `source/samples/sine_A3_single_cycle.wav` is copied into the probe.
- `plugnscript/halionbridge_sfz_pan_implementation_generator.cxx` emits note 57
  at velocity 127.
- `halionbridge-build/*.vstpreset` are generated by the real SFZ converter and
  ready to load in HALion after the build command succeeds.

## Cases

| SFZ | Purpose |
| --- | --- |
| `000_pan_center.sfz` | Default center pan reference; generated Lua should omit `amp_pan`. |
| `001_pan_left_100.sfz` | Full left SFZ pan; generated Lua should write `amp_pan = -100`. |
| `002_pan_left_50.sfz` | Partial left SFZ pan; generated Lua should write `amp_pan = -50`. |
| `003_pan_right_50.sfz` | Partial right SFZ pan; generated Lua should write `amp_pan = 50`. |
| `004_pan_right_100.sfz` | Full right SFZ pan; generated Lua should write `amp_pan = 100`. |

## Regenerate

```powershell
build-release\halionbridge_artefacts\Release\halionbridge.exe convert sfz probes\sfz\034-pan-implementation-regression\source probes\sfz\034-pan-implementation-regression\halionbridge-build --overwrite
build-release\halionbridge_artefacts\Release\halionbridge.exe probes\sfz\034-pan-implementation-regression\halionbridge-build --timeout-seconds 120
```

## Measurement Loop

1. Load one `source/*.sfz` file in sforzando.
2. Put Blue Cat Plug'n Script immediately before sforzando and load
   `plugnscript/halionbridge_sfz_pan_implementation_generator.cxx`.
3. Measure left/right channel levels, stereo balance, and phase-cancel behavior
   for the sustained part of the note.
4. Replace sforzando with the matching HALion `.vstpreset`, keeping the same
   Plug'n Script instance and meter.
5. Confirm all five cases match the sforzando pan behavior. Do not manually
   alter HALion pan law for this regression.
