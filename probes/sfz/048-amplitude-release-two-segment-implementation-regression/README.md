# 048 Amplitude Release Two Segment Implementation Regression

This probe follows `047-amplitude-release-two-segment-diagnostic`.

Probe `047` verified that the curved two-segment HALion release approximation
is better than both the previous single-segment approximation and the linear
two-segment tail variant. This probe is converter-generated from source SFZ
files and verifies that `halionbridge convert sfz` now emits that mapping.

## Files

- `source/*.sfz` are the sforzando references.
- `source/samples/amp_env_loop.wav` is copied from probe `043`.
- `plugnscript/halionbridge_sfz_amp_envelope_generator.cxx` is copied from
  probe `043`.
- `halionbridge-build/*.lua` are generated by `halionbridge convert sfz`.
- `halionbridge-build/*.vstpreset` are built through HALion and ready to load.

## Cases

Each generated HALion preset should contain a positive-release amp envelope
with:

- an early release point at `0.097 * ampeg_release`,
- early level `0.35`,
- early Lua curve `-0.24`,
- a final zero point whose segment duration completes `0.604 * ampeg_release`,
- final Lua curve `-1.0`.

Explicit zero-release behavior is covered by probe `043/007` and the unit
tests; this regression focuses on the positive-release curve.

| Case | SFZ release |
| --- | ---: |
| `000` | `0.25` |
| `001` | `0.50` |
| `002` | `1.00` |
| `003` | `2.00` |

```powershell
build-release\halionbridge_artefacts\Release\halionbridge.exe probes\sfz\048-amplitude-release-two-segment-implementation-regression\halionbridge-build --timeout-seconds 120
```

## Measurement Loop

1. Put Blue Cat Plug'n Script immediately before sforzando or HALion.
2. Load `plugnscript/halionbridge_sfz_amp_envelope_generator.cxx`.
3. Compare same-number pairs:
   - `source/000_release_0p25.sfz` vs `halionbridge-build/000_release_0p25.vstpreset`,
   - `source/001_release_0p50.sfz` vs `halionbridge-build/001_release_0p50.vstpreset`,
   - `source/002_release_1p00.sfz` vs `halionbridge-build/002_release_1p00.vstpreset`,
   - `source/003_release_2p00.sfz` vs `halionbridge-build/003_release_2p00.vstpreset`.

Use the phase-inverted/null-test setup and focus on the release tail after
note-off.

## Result

Generation and HALion build completed successfully on 2026-06-14: four SFZ
files generated four build scripts, and HALion saved four `.vstpreset` files.

Generated Lua inspection confirmed the expected two release points. For
`ampeg_release=1.0`, the generated points are:

- `{ level = 0.349999994, duration = 0.0970000029, curve = -0.239999995 }`,
- `{ level = 0, duration = 0.506999969, curve = -1 }`.

Manual validation on 2026-06-14: the converter-generated presets work as well
as the hand-authored curved two-segment candidates from probe `047`.
