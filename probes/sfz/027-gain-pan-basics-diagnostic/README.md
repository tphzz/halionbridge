# 027 Gain Pan Basics Diagnostic

This controlled probe checks SFZ gain, pan, and amplitude velocity tracking
before implementing additional ConvertWithMoss/SFZv2 mappings in halionbridge.

Use sforzando as the reference and the generated HALion `.vstpreset` files as
the converted output.

## Files

- `source/*.sfz`
  - Reference SFZ files for sforzando.
- `source/samples/sine_A3_single_cycle.wav`
  - Shared sine single-cycle sample.
- `halionbridge-build/*.vstpreset`
  - HALion presets generated by halionbridge.
- `plugnscript/halionbridge_sfz_gain_pan_velocity_generator.cxx`
  - Blue Cat Plug'n Script MIDI generator.

## SFZ Cases

| File | Opcode under test | Expected reference behavior |
| --- | --- | --- |
| `000_gain_pan_baseline.sfz` | none | Baseline centered level. |
| `001_volume_minus_6.sfz` | `volume=-6` | About 6 dB quieter than baseline. |
| `002_volume_plus_6.sfz` | `volume=6` | About 6 dB louder than baseline. |
| `003_pan_left_100.sfz` | `pan=-100` | Fully or strongly left, depending on sforzando pan law. |
| `004_pan_right_100.sfz` | `pan=100` | Fully or strongly right, depending on sforzando pan law. |
| `005_amp_veltrack_0.sfz` | `amp_veltrack=0` | Velocity should not change level. |
| `006_amp_veltrack_50.sfz` | `amp_veltrack=50` | Velocity should change level less than default. |
| `007_amp_veltrack_200.sfz` | `amp_veltrack=200` | Velocity should change level more than default. |

## Generation

From the repository root:

```powershell
build-release\halionbridge_artefacts\Release\halionbridge.exe convert sfz probes\sfz\027-gain-pan-basics-diagnostic\source probes\sfz\027-gain-pan-basics-diagnostic\halionbridge-build --overwrite
build-release\halionbridge_artefacts\Release\halionbridge.exe probes\sfz\027-gain-pan-basics-diagnostic\halionbridge-build --timeout-seconds 120
```

## Manual Validation Loop

1. Load one `source/*.sfz` file in sforzando.
2. Load the matching generated `.vstpreset` in HALion 7.
3. Insert Blue Cat Plug'n Script before the sampler under test.
4. Load `plugnscript/halionbridge_sfz_gain_pan_velocity_generator.cxx`.
5. Start host transport.
6. Compare sforzando and HALion output with level meters, stereo meters,
   phase inversion, or by ear.

The generator repeats MIDI note `57` at velocities `32`, `64`, `100`, and
`127`.

## Current Expected halionbridge Result

This is a diagnostic probe. Before gain/pan implementation:

- `volume` is expected to be missing from generated HALion output.
- `pan` is expected to be missing from generated HALion output.
- `amp_veltrack` may already be close because halionbridge emits
  `Amp Env.VelocityToLevel`; this probe verifies whether that target and unit
  match sforzando.

Record each case as:

- matched,
- mismatched,
- ambiguous,
- or not testable.
