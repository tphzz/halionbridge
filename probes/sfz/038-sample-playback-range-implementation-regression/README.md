# 038 Sample Playback Range Implementation Regression

This probe validates the real converter implementation for SFZ `offset` and
`end` after probes `036` and `037`.

The converter should emit:

```lua
sample_playback = {
    sample = "...",
    offset = sfz_offset,
    finish = sfz_end,
}
```

The helper then writes HALion `SampleOsc.SampleEnd = finish + 1` before
`SampleOsc.SampleStart = offset`. Probe `037` verified that HALion's sample-end
parameter behaves as an exclusive marker while SFZ `end` is inclusive.

## Files

- `source/*.sfz` are copied from the impulse-boundary probe `037`.
- `source/samples/*.wav` are silent impulse samples with one impulse at sample
  `22049` or `44099`.
- `plugnscript/halionbridge_sfz_sample_end_boundary_generator.cxx` emits note
  57 at velocity 127.
- `halionbridge-build/*.lua` were generated by `halionbridge convert sfz`.
- `halionbridge-build/*.vstpreset` were built through HALion and are ready to
  load.

## Measurement Loop

1. Put Blue Cat Plug'n Script immediately before sforzando or HALion.
2. Load `plugnscript/halionbridge_sfz_sample_end_boundary_generator.cxx`.
3. Load each `source/*.sfz` in sforzando and the same-number generated HALion
   `.vstpreset`.
4. In a phase-reversed null-test setup, the matching converter output should
   produce no click for the same-number boundary cases:
   - `002_end_22049_impulse_22049.sfz` vs
     `002_end_22049_impulse_22049.vstpreset`,
   - `005_end_44099_impulse_44099.sfz` vs
     `005_end_44099_impulse_44099.vstpreset`,
   - `006_offset_22050_end_44099_impulse_44099.sfz` vs
     `006_offset_22050_end_44099_impulse_44099.vstpreset`.
