# 051 Amplitude Decay Implementation Regression

This probe follows `050-amplitude-decay-shape-diagnostic`.

Probe `050` identified the current converter target for static amp decay:

- default `ampeg_decay_zero=1` with nonzero sustain uses
  `ampeg_decay * (1 - sustain) * 0.5`, Lua curve `-1.0`;
- explicit `ampeg_decay_zero=0` uses `ampeg_decay * 0.604`, Lua curve `-1.0`;
- decay to sustain level `0` uses the same curved two-segment shape as the
  verified release approximation;
- release points scale their early level by the sustain level, so a sustain
  level of `0` does not create the release bump heard in probe `049`.

This probe regenerates the broader `049` amplitude-envelope fixture with the
current converter.

## Files

- `source/*.sfz` are same-number sforzando references copied from probe `049`.

## Measurement Loop

1. Put Blue Cat Plug'n Script immediately before sforzando or HALion.
2. Load `plugnscript/halionbridge_sfz_amp_envelope_generator.cxx`.
3. Compare same-number pairs from `000` through `008`.

Expected high-signal checks:

- `003_decay_sustain_40` should now follow the `050/003` style duration.
- `004_decay_zero_0` should now follow the `050/005` style duration.
- `005_delay_attack_hold` should keep delay, attack, and hold behavior while
  improving the decay stage.
- `008_sustain_zero` should no longer have the HALion release bump.

`006_start_50` may still differ because probe `049` suggested `ampeg_start`
needs a separate focused check.
