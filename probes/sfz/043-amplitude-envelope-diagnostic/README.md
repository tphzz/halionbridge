# 043 Amplitude Envelope Diagnostic

This probe validates the current converter-backed `ampeg_*` mapping against
sforzando before tuning HALion envelope-point details.

## Files

- `source/*.sfz` are same-number sforzando references.
- `source/samples/amp_env_loop.wav` is a generated one-second seamless sine
  loop.
- `plugnscript/halionbridge_sfz_amp_envelope_generator.cxx` emits deterministic
  note input for both sforzando and HALion.
- `halionbridge-build/*.lua` are generated by `halionbridge convert sfz`.
- `halionbridge-build/*.vstpreset` are built through HALion and ready to load.

## Cases

- `000_default_release`: no explicit `ampeg_*`; checks the SFZv2 default
  release generated by the converter.
- `001_release_1s`: full-level sustain with a one-second release.
- `002_attack_1s`: one-second attack, full sustain, medium release.
- `003_decay_sustain_40`: decay to `ampeg_sustain=40` with default
  `ampeg_decay_zero=1` behavior.
- `004_decay_zero_0`: same decay/sustain shape with `ampeg_decay_zero=0`.
- `005_delay_attack_hold`: delay, attack, hold, decay, sustain, and release in
  one case.
- `006_start_50`: nonzero `ampeg_start` into the attack/decay path.
- `007_release_zero`: explicit `ampeg_release=0`.
- `008_sustain_zero`: decay to zero sustain, useful for checking whether the
  envelope should free-run or stay at the sustain point.

## Measurement Loop

1. Put Blue Cat Plug'n Script immediately before sforzando or HALion.
2. Load `plugnscript/halionbridge_sfz_amp_envelope_generator.cxx`.
3. Compare same-number pairs:
   - `000_default_release.sfz` vs `000_default_release.vstpreset`,
   - `001_release_1s.sfz` vs `001_release_1s.vstpreset`,
   - `002_attack_1s.sfz` vs `002_attack_1s.vstpreset`,
   - `003_decay_sustain_40.sfz` vs `003_decay_sustain_40.vstpreset`,
   - `004_decay_zero_0.sfz` vs `004_decay_zero_0.vstpreset`,
   - `005_delay_attack_hold.sfz` vs `005_delay_attack_hold.vstpreset`,
   - `006_start_50.sfz` vs `006_start_50.vstpreset`,
   - `007_release_zero.sfz` vs `007_release_zero.vstpreset`,
   - `008_sustain_zero.sfz` vs `008_sustain_zero.vstpreset`.

Use the meter/oscilloscope/null-test setup where practical. For this probe,
attack/decay/release timing and shape matter more than exact loop tone color.

## Result

HALion build completed successfully on 2026-06-14: nine generated converter
scripts processed, nine `.vstpreset` files saved.

Conversion emitted an sfizz warning for `004_decay_zero_0.sfz` because sfizz
reports `ampeg_decay_zero` as unsupported. The halionbridge explicit-opcode
overlay still reads it, and generated Lua for case `004` keeps the full
one-second decay while case `003` uses the default shortened decay duration.

Manual validation on 2026-06-14:

- `000_default_release` matched.
- `001_release_1s` did not match. HALion was generated with a linear
  one-second release, but sforzando behaves closer to a faster curved release.
  Manual HALion editing with a shorter release around `604 ms` and a strongly
  curved release segment was closer to phase cancellation.

Decision: do not close amp-envelope release behavior from probe `043`. Use
probe `044` to isolate HALion release-curve table encoding before changing the
converter.
