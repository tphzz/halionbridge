# 042 Loop Mode Implementation Regression

This probe follows `041-loop-playback-mode-marker-diagnostic`.

Probe `041` established the HALion mapping for the tested basic loop modes.
This regression uses the real `halionbridge convert sfz` output rather than
hand-authored HALion Lua.

## Files

- `source/*.sfz` and `source/samples/pre_loop_tail_clean.wav` are copied from
  probe `041`.
- `plugnscript/halionbridge_sfz_loop_marker_generator.cxx` is copied from
  probe `041`.
- `halionbridge-build/*.lua` were generated by `halionbridge convert sfz`.
- `halionbridge-build/*.vstpreset` were built through HALion and are ready to
  load.

## Expected Generated Shape

- `001_loop_continuous_full_start.sfz` generates `loop.mode = "continuous"`.
- `002_loop_continuous_offset_to_loop.sfz` generates `offset = 11000`,
  `natural_end = 154099`, and `loop.mode = "continuous"`.
- `003_loop_sustain_offset_to_loop.sfz` generates `offset = 11000`,
  `natural_end = 154099`, and `loop.mode = "sustain"`.
- The copied helper maps continuous loops to HALion sustain loop mode `1` and
  sustain loops to mode `4`.
- The copied helper initializes `SampleOsc.SampleEnd` from `natural_end` before
  writing `SampleOsc.SampleStart` for offset-only regions.
- The copied helper does not derive `SampleOsc.SampleEnd` or
  `SampleOsc.ReleaseStart` from loop points.

## Measurement Loop

1. Put Blue Cat Plug'n Script immediately before sforzando or HALion.
2. Load `plugnscript/halionbridge_sfz_loop_marker_generator.cxx`.
3. Compare same-number SFZ/HALion pairs:
   - `000_no_loop_full_start.sfz` vs `000_no_loop_full_start.vstpreset`,
   - `001_loop_continuous_full_start.sfz` vs
     `001_loop_continuous_full_start.vstpreset`,
   - `002_loop_continuous_offset_to_loop.sfz` vs
     `002_loop_continuous_offset_to_loop.vstpreset`,
   - `003_loop_sustain_offset_to_loop.sfz` vs
     `003_loop_sustain_offset_to_loop.vstpreset`.

Known residual risk: the amplitude release envelope is still not fully matched,
so judge loop start, loop transition, held-loop behavior, and post-note-off loop
mode semantics separately from release-level curve parity.

## Result

HALion build completed successfully on 2026-06-14: four generated converter
scripts processed, four `.vstpreset` files saved. Manual sforzando/HALion
validation on 2026-06-14 reported that the same-number pairs match great.

Decision: close the tested basic loop-mode implementation for
`loop_continuous`, `loop_sustain`, offset-to-loop playback, natural sample-end
initialization, and no loop-derived sample-end truncation.
